## 📚 Detailed Step-by-Step Installation Guide

### Prerequisites
- Node.js installed
- npm (Node Package Manager) installed
- Git installed
- VS Code or similar IDE

### Step 1: Project Creation
1. Open terminal in your desired directory
2. Create new Vite project:
```bash
npm create vite@latest portfolio-milton -- --template react-ts
cd portfolio-milton
```
3. Install initial dependencies:
```bash
npm install
```

### Step 2: Tailwind CSS Setup
1. Install Tailwind CSS and its dependencies:
```bash
npm install tailwindcss @tailwindcss/vite
```

2. Update src/index.css with Tailwind import:
```css
@import "tailwindcss";
// ...rest of your CSS
```

### Step 3: TypeScript Configuration
1. Install TypeScript node types:
```bash
npm install -D @types/node
```

2. Update tsconfig.json:
```json
{
  "files": [],
  "references": [
    {
      "path": "./tsconfig.app.json"
    },
    {
      "path": "./tsconfig.node.json"
    }
  ],
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

3. Update tsconfig.app.json:
```json
{
  "compilerOptions": {
    // ...existing compilerOptions
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  },
  "include": ["src"]
}
```

### Step 4: Vite Configuration
1. Update vite.config.ts:
```typescript
import path from "path"
import tailwindcss from "@tailwindcss/vite"
import react from "@vitejs/plugin-react"
import { defineConfig } from "vite"

export default defineConfig({
  plugins: [react(), tailwindcss()],
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "./src"),
    },
  },
})
```

### Step 5: shadcn/ui Setup
1. Initialize shadcn/ui:
```bash
npx shadcn@latest init
```

2. During initialization, select the following options:
   - Style: Default (you can customize later)
   - Base color: Neutral (or your preference)
   - Global CSS file location: src/index.css
   - Components location: @/components
   - Utilities location: @/lib/utils
   - Include special characters support: Yes
   - Use CSS variables: Yes
   - Use tailwind.config.ts: Yes
   - Enable automatic class sorting: Yes

3. Add your first component (button):
```bash
npx shadcn@latest add button
```

### Step 6: Verify Installation
1. Update src/App.tsx to test the button component:
```typescript
import { Button } from "@/components/ui/button"

function App() {
  return (
    <div className="flex min-h-svh flex-col items-center justify-center">
      <Button>Click me</Button>
    </div>
  )
}

export default App
```

2. Start the development server:
```bash
npm run dev
```

### Step 7: Project Structure Verification
Ensure your project structure matches:
```
├── components.json
├── package.json
├── tsconfig.json
├── tsconfig.app.json
├── vite.config.ts
├── public/
│   └── vite.svg
└── src/
    ├── App.tsx
    ├── index.css
    ├── main.tsx
    ├── components/
    │   └── ui/
    │       └── button.tsx
    └── lib/
        └── utils.ts
```

### Common Issues and Solutions
1. **Path Alias Issues**
   - Ensure both tsconfig files have correct path aliases
   - Verify vite.config.ts has matching alias configuration

2. **Styling Issues**
   - Check if Tailwind CSS is properly imported
   - Verify index.css has correct imports
   - Clear browser cache if needed

3. **Component Import Issues**
   - Verify path aliases in import statements
   - Check case sensitivity in file paths
   - Ensure all dependencies are installed
